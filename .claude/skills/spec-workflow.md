# Skill: Spec Workflow
Antes de implementar cualquier feature:
1. Existe una spec en specs/XXXX-nombre/spec.md con requisitos EARS,
   escenarios BDD y casos de test de tabla con datos del manual.
2. Los tests de tabla están escritos ANTES del código (TDD).
3. Valida conmigo que cada test es fiel al manual antes de hacer verde.
4. La IA implementa; el humano valida specs y fixtures.
Fuente de verdad: el manual Football RPG junto con el PRD que se encuentran en /doc.
